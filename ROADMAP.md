# Приложение за следене на наеми — Project Roadmap

> Работно име: **RentTrack** *(подлежи на промяна — избери име преди публикуване и провери дали е свободно в App Store / Google Play)*
> Версия на документа: 1.1 *(добавена Фаза 4.5 — консолидация, UX и подготовка за публикуване)*
> Платформи: iOS + Android (един код)

---

## 1. Контекст и проблем

Дребните и индивидуалните собственици на имоти под наем рядко имат удобен инструмент за следене на наемите си. Повечето използват тетрадка, Excel или просто паметта си, което води до пропуснати плащания, забравени падежи и липса на ясна картина за дохода.

Съществуващите приложения на пазара са основно насочени към американския пазар, сложни са, скъпи и не отчитат местните особености (език, валута, начин на работа на българския хазяин). Това оставя реална, недостатъчно покрита ниша.

**Накратко:** просто, фокусирано мобилно приложение, което прави следенето на наеми лесно и спокойно — без излишна сложност.

---

## 2. Цел на проекта

**Дългосрочна визия:** сериозен продукт, публикуван в App Store и Google Play, който дребни собственици използват ежедневно за управление на имотите си.

**Стратегия на разработка (важно):**
1. Първо — работещо приложение **за лично ползване** (валидира, че решава реален проблем).
2. После — отваряне **за други потребители** (вход, облак, синхронизация).
3. Накрая — **публикуване и монетизация**.

Това разделение е съзнателно: не се строи „за всички" от първия ред код. Първо нещо, което работи за самия теб.

**Как изглежда успехът:**
- v1: ползвам приложението сам, вместо таблица/тетрадка.
- v2: 5–10 души извън мен го ползват редовно и дават обратна връзка.
- v3: публикувано в двата магазина с работещ абонаментен модел.

---

## 3. Целеви потребители

- Дребни собственици с 1–20 имота (апартаменти, гаражи, земя, офиси).
- Хора, които не искат сложен счетоводен софтуер, а просто ред и спокойствие.
- Първоначален фокус: български пазар (език, евро/лев, локални навици).

## 4. Обхват

### В обхвата на v1 (MVP) — базовото
- Добавяне на имоти от различен тип (апартамент, гараж, земя, друго).
- Статус на всеки имот: свободен / под наем.
- Данни за наемател: име, телефон, сума на наема, ден за плащане.
- Записване на плащания и история на плащанията.
- Просто табло за преглед (доход, събрано/дължимо, заетост).

### В обхвата на следващи версии
- Локални известия (напомняния за падеж, аларми за неплатено).
- Месечни и годишни справки + PDF разписки.
- Проследяване на депозити и разходи (нетен доход).
- Бележки и документи към имот/наемател.
- Вход, облак и синхронизация (многопотребителски режим).
- Изпращане на напомняния директно към наемателя.

### Извън обхвата (съзнателно НЕ правим в началото)
- **Реална обработка на онлайн плащания в приложението** — тежка регулация (PCI), такси и отговорност. В началото само *записваме* плащания, не ги *обработваме*.
- Пълна счетоводна / данъчна интеграция.
- Едновременно уеб + мобилно приложение.
- Сложна автоматизация (банкови извлечения, OCR на договори) — много по-късно, ако изобщо.

## 6. Модел на данните

Сърцето на проекта. Три основни таблици + две по-късно. Наемателят се пази отделно от имота през „наемно правоотношение" (lease), за да остава история при смяна на наемателите.

### `properties` (имоти)
| Поле | Тип | Описание |
|---|---|---|
| id | UUID / int | Първичен ключ |
| type | text | apartment / garage / land / office / other |
| name | text | Етикет, напр. „Ап. 3, ул. Осма" |
| address | text | Адрес (по избор) |
| status | text | free / rented / unavailable |
| notes | text | Свободни бележки |
| created_at | datetime | |

### `tenants` (наематели)
| Поле | Тип | Описание |
|---|---|---|
| id | UUID / int | Първичен ключ |
| name | text | Име |
| phone | text | Телефон |
| email | text | По избор |
| notes | text | Бележки |
| created_at | datetime | |

### `leases` (наемни правоотношения)
| Поле | Тип | Описание |
|---|---|---|
| id | UUID / int | Първичен ключ |
| property_id | FK → properties | Кой имот |
| tenant_id | FK → tenants | Кой наемател |
| rent_amount | number | Сума на наема |
| currency | text | EUR (основна) / BGN |
| payment_day | int (1–31) | Ден за плащане в месеца |
| start_date | date | Начало |
| end_date | date / null | Край (null = активен) |
| deposit_amount | number | Депозит |
| status | text | active / ended |
| notes | text | |

### `payments` (плащания)
| Поле | Тип | Описание |
|---|---|---|
| id | UUID / int | Първичен ключ |
| lease_id | FK → leases | Кое правоотношение |
| period | text (YYYY-MM) | За кой месец е плащането |
| amount | number | Платена сума |
| paid_date | date | Дата на плащане |
| status | text | paid / partial / pending |
| method | text | в брой / банка / друго |
| notes | text | |

### Бъдещи таблици
- `expenses` — разходи по имот (данъци, ремонти, поддръжка) → нетен доход.
- `documents` — снимки/файлове към имот или наемател (договор, електромер). **Внимание: GDPR при чужди лични документи.**

### Изчисляеми (derived) стойности — не се пазят като полета
- **Заетост:** брой имоти със статус `rented` спрямо общия брой.
- **Дължимо за месец:** активни leases без `paid` плащане за текущия период.
- **Неплатено / просрочено:** ако периодът мине след `payment_day` без записано плащане.

## 7. Архитектура

**v1 (local-first):** всичко на телефона в SQLite. Без сървър, без интернет, без регистрация. Бързо за разработка и напълно достатъчно за лично ползване.

**v2 (cloud):** добавя се Supabase — вход с email/парола, данните на всеки потребител отделени (row-level security), синхронизация между устройства. Локалната база остава като кеш за работа офлайн.

**Примерна структура на папките (Expo Router):**
```
/app            # екрани и навигация (файлова)
  /(tabs)       # табло, имоти, наематели, настройки
  /property/[id]
  /tenant/[id]
/components     # преизползваеми UI компоненти
/db             # схема, миграции, заявки
/lib            # помощни функции (дати, валута, изчисления)
/hooks          # custom hooks
/store          # състояние (Zustand)
/notifications  # логика за известия
```

## 8. Логика на известията

- За всеки активен lease се насрочва локално известие **N дни преди** `payment_day` всеки месец („Наемът за [имот] е до [дата]").
- При отваряне на приложението (или ежедневна фонова проверка) се проверява всеки активен lease: ако текущата дата е минала `payment_day` и няма `paid` плащане за периода → маркира се като просрочено и се показва аларма/значка.
- Известията са **локални** (без нужда от сървър) — затова работят и в v1.

---

## 9. Локална особеност: валута (евро / лев)

България е във валутния съюз — от 1 януари 2026 г. еврото е официалната валута (фиксиран курс 1 EUR = 1.95583 BGN). Двойното обозначаване на цени в лева и евро е задължително до 8 август 2026 г.

**Извод за приложението:** гради на **евро като основна валута**, но добави опция за паралелно показване в лева през преходния период. Дребна функция, която показва, че приложението е направено за българския пазар.

Приложението обработва лични данни (имена, телефони, плащания, евентуално документи на наематели). При отваряне за други потребители това става сериозно:
- Декларация за поверителност и условия за ползване.
- Ясно правило как се събират, пазят и трият данните.
- Минимизиране: събирай само каквото е нужно.
- Право на изтриване на данни при поискване.
- Внимавай особено със снимки на лични документи (Фаза „documents").

## 13. Монетизация (по-късно)
